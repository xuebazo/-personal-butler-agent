## 零、需求本质重述

你要的不是一个"管家"，是一个**数字孪生 (Digital Twin)**。

逐层拆解你说的：

| 你说的 | 本质需求 | 技术对应 |
|--------|---------|---------|
| "蒸馏我的数据，模仿我的语气、做事规律" | 风格克隆 + 行为模式建模 | 写作风格指纹 + 决策模式提取 |
| "在一定层面上成为我，帮我完成工作" | 代理决策 + 任务执行 | 基于你历史决策训练的偏好模型 + LLM agent |
| "比我更懂我" | 发现你自己没意识到的模式 | 跨时间维度分析，揭示隐含规律 |
| "全方案覆盖，针对制定最符合我" | 个性化方案生成，不是模板匹配 | 你的约束条件 + 偏好函数 → 方案排序 |
| "预测情绪衰减机制" | 以时间/事件为轴的情绪和精力模型 | 时序数据追踪 + 触发事件关联 |
| "日常贴近我，工作时成为我" | 双模式切换 | 同伴模式 vs 代理模式 |

**一句话总结：做一个随时间演化、越来越像你的数字副本。**

#### 模块 1: 人格引擎 (Identity Engine)

**核心问题：你是谁？你怎么想？**

这是整个系统的基础层，其他模块都依赖它。

**数据来源：**
| 来源 | 揭示什么 | 当前状态 |
|------|---------|---------|
| about-me.md | 显性自我认知 | 已有 |
| persona-prompt.md | 期望的交互风格 | 已有 |
| 聊天蒸馏 (distill.py) | 隐含偏好、思维模式 | 已有 |
| 对话洞察日志 | 逐次对话学到的新认知 | 已有 |
| **写作样本** | **语气指纹、表达节奏** | **缺少** |
| **决策记录** | **选择逻辑、权衡权重** | **缺少** |
| **日程/行为数据** | **精力周期、时间偏好** | **缺少** |

**人格模型的维度（建议从这些角度建模）：**

```
人格画像 = {
  认知风格: {
    信息处理: "先框架后细节" | "从具体到抽象" | ...
    决策模式: "数据驱动" | "直觉优先" | "权衡清单" | ...
    风险偏好: "激进" | "保守" | "情境依赖" | ...
  },
  沟通风格: {
    语气指纹: "简洁直接" | "委婉" | "自嘲" | ...
    节奏特征: "短句" | "长段落" | "先结论后论证" | ...
    情绪表达: "外露" | "内敛" | "行动化" | ...
  },
  行为模式: {
    精力曲线: {早晨: 高, 午后: 低, 夜晚: 中},
    注意力偏好: "深度单线程" | "多任务并行" | ...
    触发敏感: ["被催促" → "焦虑", "不明确" → "拖延", ...]
  },
  价值观权重: {
    效率: 0.8,
    自主: 0.7,
    美感: 0.4,
    ...
  }
}
```

**关键设计决策：人格画像应该是显式结构化数据 vs 隐式 prompt 描述？**

建议：**结构化为主，prompt 为辅。** 结构化数据可以做量化分析、趋势对比；prompt 文本用于 LLM 交互。

#### 1.1 人格画像的生成方式：Profiler 子系统

**核心问题：怎么从原始素材中提取结构化人格画像？用开源库、爬书、还是自己做？**

**调研结论（2026-05-28）：**

| 路径 | 代表方案 | 结论 |
|------|---------|------|
| 开源库 | OCEAN-AI (`pip install oceanai`)，学术级，基于传统 NLP 特征提取 | 不可用。中文不支持，产出的只是 OCEAN 五个分数，不是完整人格画像 |
| 爬专业书 | 人格心理学教材、论文 | 不必要。LLM 已经内化了 Big Five、MBTI、依恋理论等心理学框架 |
| LLM + Prompt | 2025 年学术界一致结论——最有效的方法 | 采用 |

2025 年论文 `MBTI-in-Thoughts` 和 `Psychologically Enhanced AI Agents` 验证：**LLM + 好的 prompt 工程 = 专业级人格分析**，支持 Big Five / HEXACO / Enneagram 等多框架，无需微调。

#### 1.2 Profiler 架构：四轮分析流水线

```
第 1 轮：素材整理（不做分析，只聚合归类）
  输入: about-me.md + 对话日志 + distill 洞察 + 写作样本
  输出: 按主题聚合的原始素材段落

第 2 轮：分维度独立分析（每维度一个独立 prompt，互不污染）
  维度 A: 认知风格    → "如何处理信息？如何做决策？"
  维度 B: 沟通指纹    → "怎么说话？句式？节奏？禁忌词？"
  维度 C: 情绪模式    → "触发事件 → 反应 → 衰减曲线？"
  维度 D: 价值观权重  → "什么重要？冲突时优先什么？"
  维度 E: 行为规律    → "精力周期？工作习惯？社交模式？"
  维度 F: 隐含盲区    → "他可能没意识到的模式？"
  
  每个维度输出: 3-5 个模式 + 原文引用证据

第 3 轮：交叉验证
  输入: 第 2 轮 6 个维度的完整输出
  输出: 维度间的一致性/矛盾/互补关系
  例如: "价值观说重视效率，但行为数据显示深夜效率低下 → 矛盾"

第 4 轮：生成最终画像
  输入: 第 3 轮的交叉分析
  输出: 结构化 personality-profile.yaml
```

**Prompt 设计原则：**
- 每个 prompt 只分析**一个维度**，维度之间互不污染
- 不是 "请分析这个人的性格"，而是 "以下是这个人的对话记录。请聚焦于【决策模式】这一维度，找出所有证据，归纳出 3-5 个模式，每个附带 1-2 个原文引用"
- 每个维度输出要求：模式描述 + 证据引用 + 置信度

**代码结构：**

```
scripts/
├── profiler.py              ← 新增 (~150行)
│   ├── collect_materials()  ← 从各数据源聚合文本
│   ├── analyze_dimension()  ← 单维度分析（调用 LLM）
│   ├── cross_validate()     ← 交叉验证
│   └── generate_profile()   ← 生成最终 YAML
│
└── profiler_prompts.py      ← 新增 (~50行)
    ├── COGNITIVE_STYLE_PROMPT

    ├── BEHAVIORAL_RHYTHM_PROMPT
```

#### 模块 2: 记忆系统 (Memory System)

**核心问题：你经历了什么？**

当前系统只有一个扁平 ChromaDB collection。需要分层：

```
记忆层级：
┌────────────────────────────────────┐
│ L1 核心记忆（永久）                 │
│   about-me, 核心价值观, 不变的原则   │
│   更新频率：几乎不变                  │
│   存储：Chromadb + 权重最高           │
├────────────────────────────────────┤
│ L2 情景记忆（长期）                  │
│   重要事件、重大决策、转折点          │
│   更新频率：月/年                     │
│   存储：带时间戳的结构化条目           │
│   自然衰减：时间越久权重越低           │
├────────────────────────────────────┤
│ L3 工作记忆（中期）                  │
│   近期关注、进行中的项目、当前状态      │
│   更新频率：周                        │
│   存储：带时间衰减函数                 │
│   容量限制：最近 N 条                  │
├────────────────────────────────────┤
│ L4 会话记忆（短期）                  │
│   当前/最近对话                       │
│   更新频率：实时                      │
│   存储：conversation history          │
│   生命周期：本次会话                   │
└────────────────────────────────────┘
```

**记忆衰减函数：**
```
记忆强度 = 初始强度 × e^(-λt) × 事件重要性系数 × 重温次数
```
- 不是所有记忆都同等重要
- 被反复提及的信息应该强化
- 情绪峰值事件（emo 崩溃、重大成就）衰减更慢

#### 模块 3: 状态追踪 (State Tracker)

**核心问题：你现在怎么样？这可能是最被低估但最关键的模块。**

你要的"情绪衰减机制预测"就在这里。

**追踪维度：**

```
状态快照 = {
  时间戳: "2026-05-28 22:30",
  情绪: {
    效价: -0.3 ~ +0.7,        // 积极-消极
    唤醒: 0.2,               // 兴奋-平静
    主导情绪: "疲惫但平静",
    触发事件: ["连续工作8h", "未吃晚饭"]
  },
  精力: {
    主观评分: 4/10,
    认知带宽: "低",           // 适合简单任务
    注意力: "分散"
  },
  环境: {
    时段: "深夜",
    天气: 雨天,
    社交状态: "独处"
  }
}
```

**数据采集方式（隐私优先，渐进式）：**
| 阶段 | 方式 | 隐私友好度 |
|------|------|-----------|
| V1 | 用户主动输入 emoji/评分 | 最高 |
| V2 | 对话中自然流露的情绪分析 | 高 |
| V3 | 从工作产出中推断（代码提交频率、消息回复速度） | 中 |

**预测模型要做的事：**
```
输入: 当前状态 + 时间窗口 + 日程事件
输出: {
  情绪预测曲线: "周三下午可能进入低点",
  精力预测: "连续3天高强度后周五会崩",
  干预建议: "建议周四安排缓冲",
  沟通适配: "现在不要提复杂决策，只说结论"
}
```

**实现可行性：**
- 规则引擎 → 足够起步。如"深夜 + 工作 > 6h → 精力低，避免复杂讨论"
- 简单时序模型 → 积累数据后用 ARIMA/LSTM
- 关键在于**轻量启动**，不要一开始就追求精度

#### 模块 4: 学习与演化系统 (Evolution Engine)

**核心问题：你正在变成谁？系统怎么跟上？**

这是让系统"越用越像你"的关键。

**三个学习回路：**

```
回路 1: 即时学习（每句话）
  负反馈 → 立即修正 → 写入偏好日志
  正反馈 → 记录"这次做得对" → 强化模式
  当前: 已有 bad 命令

回路 2: 会话学习（每次对话结束）
  从对话中提取新洞察 → 更新人格画像
  标记过时信息 → 衰减旧记忆
  当前: 已有 save_session_insights

回路 3: 周期性反思（每周/月）
  汇总本周所有洞察 → 提炼更高层次的模式
  对比历史 → 发现变化趋势
  "三个月前你喜欢A，现在偏向B，为什么？"
  当前: 缺少
```

**演化追踪的价值：**
- 知道"2026年3月时的你"和"2026年5月时的你"有什么不同
- 可以对历史状态的你进行回访——"那时候我会怎么想？"
- 这个功能本身就是你"比你自己更懂你"的来源

## 三、双模式设计：同伴 vs 代理

你提的 "日常贴近我，工作时成为我" 是整个系统最核心的使用界面。

### 同伴模式（日常）

```
目标: 陪你、懂你、帮你理清思路
行为:
  - 语气: 你舒服的语气（由人格引擎定义）
  - 主动性: 可以主动关心（"今天看起来很累？"）
  - 边界: 不替代决策，提供分析和视角
  - 反馈: 可以指出你的盲区（"你上次也这么说，但结果..."）
触发场景: 聊天、反思、情绪低落、日常讨论
```

### 代理模式（工作）

```
目标: 成为你，替你产出
行为:
  - 语气: 你的语气（更正式/专业版）
  - 主动性: 完成任务为主，不需要情感交流
  - 边界: 在明确授权范围内行动
  - 输出: 邮件、方案、回复、总结——以你的身份
触发场景: 写邮件、写方案、回复消息、做总结
```

**切换信号识别：**
```
同伴 → 代理:
  - 用户说"帮我写个..."
  - 用户说"替我给XX回复"
  - 检测到明显的任务型输入

代理 → 同伴:
  - 任务完成
  - 用户开始闲聊
  - 用户表达情绪/感受
```

## 四、你最关心的：情绪衰减预测

拆开看，这不是一个功能，是一个**子系统**。实现路径：

### 第一阶段：手动输入 + 规则（现在就能做）

```
每天 2-3 次弹窗/提醒：
  "现在状态？😄🫱😐😫"
  可选备注："刚开完3小时会"

数据积累 2 周后 → 规则引擎：
  if 连续高强度会议 > 3h and 时段 == 下午:
    预测：精力低谷
    行为：切换到"轻量模式"，减少认知负担
```

### 第二阶段：被动感知 + 统计分析（3-6 个月数据积累后）

```
从对话中自动检测情绪信号：
  - 回复长度突然变短 → 可能累了
  - 用词语气变化 → 情绪波动
  - 每次回复间隔变长 → 注意力下降
  
建立基线：你的正常波动范围
检测偏离：今天和往常不一样
```

### 第三阶段：预测 + 主动干预（6 个月以上）

```
预测：
  "根据过去 3 个月模式，你月底周二容易焦虑"
  "你这次会议安排太密，周四下午大概率状态崩"

干预：
  "要不要把下午的会挪到明天？"
  "现在你不需要做决定，先休息"
  "上周这个时间你也很难受，但周六就好了"
```

**关键：不要一上来就追求第三阶段。数据不够时预测是噪音。先用好第一阶段。**

## 七、开放问题

1. **Profiler 分析维度确认**：6 个维度（认知/沟通/情绪/价值观/行为/盲区），是否有遗漏？沟通指纹中的 "禁止词" 你觉得对吗？有没有补充？

2. **Profiler 素材准备**：你准备好提供能体现"你是谁"的文字素材了吗？包括（可以脱敏）：
   - 代表性的写作样本（工作邮件、随手记、社交发言）
   - 更多的 about-me 内容
   - 任何你觉得 "对，这就是我" 的瞬间记录

3. **personality-profile.yaml 替代 persona-prompt.md**：同意用结构化 YAML 彻底替代纯文本 prompt 吗？

4. **数据采集的边界**：情绪打卡 OK 吗？还是更偏好从对话中自然提取？日程/日历数据你愿意接入吗？

5. **代理模式的权限边界**：什么任务可以"代替你"（写邮件），什么不行（做重要决策）？需要明确。
